## Successes

1. **⭐ We finally have a design validator.** The standout win (ifero): 16-5 stands up a web Storybook (react-native-web) with **Chromatic** visual-regression on PRs, built on 16-3's design-in-code contribution workflow and 16-4's portable DTCG tokens. Design/UI changes are now reviewable without a Figma seat or a local build — the closest thing to a git-anchored, account-free design-review surface. Four stories that compose into one capability. (The Chromatic project + token is the one remaining human prereq to switch the automated visual diffs fully on.)

2. **The reliability trilogy was diagnosed, not guessed.** 16-7, 16-8, and 16-10 each open with a file:line root cause and _refute the naive theory first_. 16-8 explicitly refuted "Supabase client isn't ready" and pinned the `autoTriggeredRef` latch-on-first-attempt (`useCloudSync.ts:113`); 16-7 traced the real CI failure (run `27338428755`) to `expo prebuild` regenerating `versionCode 1`; 16-10 located the unguarded `getSession()` network await on the boot path (`app/_layout.tsx:311`). Fixes were root-cause + defense-in-depth, not symptom patches.

3. **Adversarial review that _executes_ caught what green gates missed — twice.** In 16-5 the reviewer drove a **real browser** and found every story crashed on web (the Unistyles side-effect module was never imported → `setTheme` before `configure`) — build and jest were both green. In 16-10 the reviewer traced the **installed** `@supabase/auth-js@2.105.1` and empirically **disproved the locked ADR's premise**: `INITIAL_SESSION` is _not_ synchronous for an expired token (the SDK network-refreshes first), which would have regressed AC4 (reinstalled signed-in user bounced to `/welcome`). The `hasPersistedSession()` probe preserved the ADR's intent with a corrected mechanism.

4. **Root-cause fixes that stop recurrence.** 16-9 didn't just relocate the fat screens — it **implemented the documented-but-missing route-file ESLint rule** (`no-restricted-imports` banning hooks in `app/**/*.tsx`), so the leak "cannot silently recur," and it caught that the architecture doc's own rule snippet was buggy for flat config. 16-7 put `versionCode` in `app.config.ts` so it survives prebuild instead of hand-editing gitignored generated files.

5. **Tight-scope discipline held under pressure.** 16-8 and 16-10 both deferred "consider" items with explicit `flag-don't-fix` rationale, keeping diffs reviewable — then 16-8's dedup was consciously completed as a stakeholder-requested efficiency pass (module-level singleton `useCloudSync` store).

6. **Zero production incidents** across a 155-file source diff that swapped the styling engine and relocated four screens — while the suite grew ~172 tests and the 80% coverage gate held.

## Challenges

> Psychological safety: these are systems/process observations, not individual performance.

1. **Device/hardware validation debt is compounding.** Sprint 16 added five ifero-owned, sandbox-impossible validation gates: 16-1 (on-device light/dark sweep + dev-client build), 16-2 (prod Sentry smoke test), 16-8 (cold-open sync on device), 16-10 (airplane-mode cold start) — plus Epic 9's still-open watch validation. Same root as Epic 9 action #2 ("green tests aren't a validated app"), now five items instead of one.

2. **"Green tests ≠ works" — the sandbox blind spot made real.** 16-1's nav-bar flash was found by ifero _on device, after_ code-review + QA both passed (the Unistyles babel plugin remaps `Pressable` → shadow-registry churn, regressing even unedited files). 16-5's two blockers slipped past build + jest. The CI sandbox can't run native/browser/release paths, so a class of defect only appears at the device gate.

3. **Stale story assumptions when work is sequenced/parallel.** 16-4's "Verified constraints" (`tailwind.config.js`, `TAILWIND_*` exports, typography tuple) **no longer existed** because 16-1 landed first and deleted them; the dev validated byte-stability against the _actual_ current API instead of the story text.

4. **Uncommitted WIP drift.** 16-3 was implemented + reviewed 2026-06-19 but **left uncommitted while `main` advanced 4 commits**, forcing a full re-verification on a fresh branch 2026-07-07 (fresh review + QA loops, 8 more findings incl. 2 MAJOR). This contributed to 5 of 9 stories merging in the final four days.

5. **Action items with dissolved ownership stall.** The 6.10 migration removed the SM (Bob) mid-sprint. The two Bob-owned Epic 9 items — sprint-record reconciliation and the Epic 5 complication — are exactly the two that did not cleanly close. Retired roles orphan their commitments unless explicitly re-homed.

6. **Recurring carry-forward: Sentry prod verification.** Epic 9 action #4 ("verify Sentry fires in production") is carried for a _second_ consecutive sprint — 16-2 wired everything (incl. #140 CI symbols) but the release-build smoke test remains unrun (needs a device/CI release build).

7. **Coverage blind spot exactly where the risk is.** `jest.config.js` `collectCoverageFrom` covers only `core/**` + `features/**`; `shared/**` (where `useBootAuthGate`, `client.ts`, `useCloudSync` live) is outside the 80% gate. Surfaced independently in **both** 16-4 and 16-10 → drafted as 16-13.
